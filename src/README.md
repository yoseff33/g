# نظام فزاع المالي والاستثماري المتكامل (Fazza ERP) 🇸🇦

منظومة تخطيط موارد مالية (ERP) متكاملة ومصممة خصيصاً لتلبية أعلى المعايير المهنية للمنشآت المالية والاستثمارية العقارية في المملكة العربية السعودية. النظام يعمل بدقة إنتاجية 100% دون أي محاكاة، وبمزامنة حية مباشرة مع قواعد بيانات Supabase.

---

## 🌟 أبرز مميزات المنظومة (11 وحدة مالية متكاملة)

1. **لوحة التحكم الحية (Real-time Dashboard):** مؤشرات أداء مالية دقيقة (KPIs)، إجمالي المحافظ الاستثمارية، النقدية المتاحة بالصناديق، الأرباح المحققة، ورسوم بيانية ذكية تفاعلية للميزانية وصافي العوائد.
2. **شؤون الشركاء والمستثمرين (Investors):** إدارة بيانات الشركاء، حساب الحصص، وأرشفة الهويات الوطنية مع خاصية توليد كشوف الحسابات التفصيلية الجاهزة للطباعة فوراً.
3. **متابعة العقود الاستثمارية (Contracts):** تدوين وإصدار العقود الاستثمارية، تحديد نسب الأرباح السنوية، والربط الفوري مع الحسابات الدائنة (الأستاذ العام).
4. **شجرة الحسابات المحاسبية (Chart of Accounts):** هيكل محاسبي متكامل متوافق مع الهيئة السعودية للمراجعين والمحاسبين (SOCPA)، يدعم الهرمية المتداخلة والتمييز التلقائي لحسابات الأستاذ والحسابات الترحيلية.
5. **قيود اليومية العامة (Journal Entries):** واجهة تدوين محاسبي مزدوج مع تحقق فوري على مستوى الواجهة والـ Database بضرورة تساوي أطراف القيد (SUM Debit = SUM Credit).
6. **سندات القبض والصرف (Receipts & Payments):** تدوين الحركات النقدية والبنكية مع أتمتة كاملة لتوليد وترحيل قيود السداد المزدوجة للأستاذ العام بمجرد حفظ السند.
7. **توزيع أرباح المستثمرين (Profit Distribution):** معالج احتساب مالي متقدم يقوم بتوزيع أرباح الشركاء بناءً على نسبة الأوزان الاستثمارية وعقودهم السارية وترحيلها مباشرة من حساب الأرباح المحتجزة إلى حساب الأرباح المستحقة.
8. **ميزان المراجعة (Trial Balance):** قائمة ختامية تفصيلية تستعرض الأرصدة الافتتاحية، حركات الفترة، والأرصدة النهائية للأستاذ العام.
9. **قائمة الدخل الشامل (Income Statement):** قياس فوري لأداء المنشأة وصافي أرباحها من خلال مقابلة الإيرادات التشغيلية بالمصروفات العمومية.
10. **الميزانية العمومية (Balance Sheet):** قياس متوازن للمركز المالي يثبت صحة المعادلة المحاسبية (الأصول = الالتزامات + حقوق الملكية).
11. **سجل الرقابة والتدقيق الأمني (Audit Logs):** سجل تاريخي معزز بسياسات RLS وقواعد حماية لا يمكن مسحه أو تعديله، يوثق فوارق البيانات قبل وبعد كل عملية تعديل أو حذف.

## 🏗️ البنية البرمجية والربط السحابي (Database Architecture)

النظام مبني على هيكل أمني صارم ومحمي بسياسات Row Level Security (RLS) ومؤتمت بالكامل عبر قواعد بيانات PostgreSQL:

* **العلاقة الحسابية المزدوجة:** ترتبط البنود بـ `journal_items` عبر جدول قيود اليومية `journal_entries`.
* **صمام أمان الأرصدة (`trg_check_journal_balance`):** مُطلق مدمج يمنع ترحيل أي قيد محاسبي لا يتساوى فيه طرفي القيد (المدين والدائن) تماماً، مما يضمن استحالة حدوث خلل محاسبي.
* **التتبع والتدقيق الآلي (`audit_logs`):** يتم التقاط كافة تعديلات وحذف وتحديثات الجداول وتدوين قيمها السابقة والجديدة بصيغة `JSONB` آلياً لضمان النزاهة المالية.

## 🚀 تشغيل وتجهيز النظام محلياً

### 1. المتطلبات الأساسية
* وجود بيئة تشغيل **Node.js 18+**.
* حساب على **Supabase**.

### 2. إعداد المتغيرات البيئية
قم بإنشاء ملف `.env` في المجلد الجذري للمشروع وأضف المفاتيح المستخرجة من لوحة تحكم Supabase الخاصة بك:

```env
VITE_SUPABASE_URL="https://your-project-id.supabase.co"
VITE_SUPABASE_ANON_KEY="your-anon-public-key"
```

### 3. تنصيب الحزم والتشغيل
افتح منفذ الأوامر في مجلد المشروع ونفذ الآتي:

```bash
# تنصيب الحزم المحملة
npm install

# تشغيل بيئة التطوير الحية على المنفذ 3000
npm run dev
```

### 4. بناء نسخة الإنتاج الفعلية
لبناء ملفات النظام الختامية الجاهزة للرفع على خوادم الاستضافة:

```bash
npm run build
```
