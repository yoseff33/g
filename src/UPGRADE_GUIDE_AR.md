# ترقية فزاع ERP — الإصدار الاحترافي الأول

## الميزات المضافة

- مركز تحصيل وسداد موحد.
- دعم السداد الجزئي والكامل مع رقم مرجع وطريقة دفع.
- سجل أنشطة التحصيل: واتساب، وعد سداد، ملاحظات، ودفعات.
- بوابة مستثمر تعرض العمليات والتحصيل والمتبقي مع طباعة وتصدير CSV.
- محرك توزيع عادل يرتب المستثمرين حسب الرصيد، الدور، الحمل الحالي والحد الأعلى.
- مركز موافقات للعمليات المالية الحساسة.
- عداد تنبيهات للاستحقاقات والطلبات المعلقة.
- تقسيم كود الشاشات Lazy Loading لتحسين سرعة التحميل.
- Error Boundary لمنع تعطل النظام كاملًا عند خطأ شاشة واحدة.
- إصلاح تحذير CSS وتكرار className واستيراد X.
- عنوان وأيقونة ووصف احترافي للموقع.

## 1. ترقية قاعدة Supabase

خذ نسخة احتياطية من قاعدة البيانات، ثم:

1. افتح Supabase Dashboard.
2. انتقل إلى SQL Editor.
3. افتح الملف `database/professional_upgrade.sql` من المشروع.
4. انسخ محتواه كاملًا واضغط Run.

الملف يستخدم `IF NOT EXISTS` قدر الإمكان، لكنه يجب تشغيله أولًا في بيئة اختبار عند وجود بيانات إنتاج مهمة.

## 2. متغيرات Vercel

في Vercel > Project > Settings > Environment Variables أضف:

- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_ANON_KEY`

لا ترفع ملف `.env` إلى GitHub. استخدم `.env.example` كمرجع فقط.

## 3. الفحص المحلي

```bash
npm install --include=optional
npm run lint
npm run build
npm run dev
```

## 4. الرفع إلى GitHub

```bash
git add .
git commit -m "upgrade Fazza ERP professional operations"
git push origin main
```

بعد نجاح الدفع سيبدأ Vercel النشر تلقائيًا.

## ملاحظات تشغيلية

- بوابة المستثمر الحالية شاشة إدارية لمعاينة حساب أي مستثمر. ربط دخول المستثمر بحسابه يتم عبر الحقل `investors.portal_user_id` وسيكون المرحلة التالية.
- مركز الاعتمادات يحتاج إنشاء الطلبات من الشاشات المالية لاحقًا؛ البنية والمراجعة والاعتماد جاهزة الآن.
- تسجيل السداد يستخدم دالة Supabase ذرية `register_installment_payment`. يوجد fallback في الواجهة للأنظمة القديمة، لكن يفضّل تشغيل ملف الترقية.
- التوزيع العادل يحفظ سجل القرار وسببه. إسناد العقد تلقائيًا للمرشح يمكن ربطه بنموذج العقد في المرحلة التالية.
